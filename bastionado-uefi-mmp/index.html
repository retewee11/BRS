¿Porque hacer un bastionado de la UEFI?

Hacer un bastionado de la UEFI es fundamental porque protege el arranque del sistema desde el nivel más bajo, evitando que malware o ataques avanzados comprometan el firmware antes de que cargue el sistema operativo.

¿Como entrar a la UEFI?

Dependiendo del equipo y el fabricante hay que pulsar una tecla distinta suelen ser teclas tipo: esc, supr, F9, etc...

El primer paso y el más importante es establecer una contraseña para la UEFI, ya que si hacemos todos los pasos que vamos a hacer pero no protegemos la UEFI con una contraseña, cualquiera podría entrar y resetear toda la configuración y no valdría de nada todo el bastionado.

Aquí lo primero que tenemos que modificar son las 2 ultimas opciones, para obligar a la hora de elegir una contraseña que te controle estas 2 cosas, no es algo obligatorio, pero evita futuros despistes:

  • Set Minimum Length: En esta opción podemos elegir el minimo de caracteres para las contraseñas en este caso, como se verá en la proxima imagén elijo minimo 12 caracteres.
  • Set Strong Password: Esta opción comprueba por tí si la contraseña que has elegido es fuerte.

Ahora vamos a asignar las contraseñas en este caso veremos muchos tipos de contraseñas distintas:

  • Supervisor Password: Esta contraseña es la usada para cada vez que se quiera entrar en la UEFI, es la más importante ya que es la base de todo, por lo tanto esta contraseña hay que establecerla.

  • System Management Password: Esta contraseña se usa para el tema empresarial, por si hay distintos roles de IT que tienen que hacer cosas distintas, que cada uno tenga su propia contraseña y solo acceda a lo que debe de acceder, en este caso no nos hará falta.
  • Power-On Password: Esta contraseña es la usada para cada vez que el ordenador se enciende, es una contraseña que pide antes de empezar a arrancar nada, esta contraseña se asignara o no dependiendo de lo que se quiera complicar el uso diario al que usa el equipo, si es un equipo critico, sin duda habría que establecerla.
  • NVMe1 Password: Esta contraseña es para acceder a discos duros concretos, si el uso del equipo es compartido y hay información sensible en disco concreto podría establecerse.

Para todo esto es super importante el uso de contraseñas distintas y que las contraseñas sean fuertes ya que, de nada vale poner una contraseña si es el nombre y apellido por ejemplo de quien lo usa.

Secure Boot es una función de la UEFI que asegura que solo el software firmado y autorizado por el fabricante puede arrancar el equipo. Su objetivo principal es impedir la ejecución de bootloaders, sistemas operativos o controladores no autorizados y, por tanto, proteger el sistema frente a malware a nivel de arranque (como rootkits).

Cuando Secure Boot está activado:

  • La UEFI verifica la firma digital de cada componente antes de cargarlo durante el arranque.
  • Si detecta un componente no firmado o manipulado, bloquea su ejecución.
  • Solo se permite arrancar sistemas y controladores que hayan sido aprobados y firmados previamente.

Este mecanismo aporta una capa de seguridad clave contra amenazas avanzadas que buscan persistir antes del sistema operativo.

Es muy importante verificar que está activo.

¿Que es el tpm 2.0?

TPM 2.0 es un chip de seguridad en la placa base que almacena y protege claves de cifrado, contraseñas y certificados digitales, ayudando a proteger el sistema contra accesos no autorizados y malware.

Comprobaciones

  • Comprobar que el TPM 2.0 este activado.
  • Comprobar que Physical Presence for Clear, esto exige presencia física para borrar el chip, añadiendo una barrera contra ataques remotos o scripts automatizados.

¿Que hacer aquí?

  1. Deshabilita arranques innecesarios
    • En "Network Boot" selecciona la opción más restrictiva (por ejemplo, desactiva PXE Boot si no usas arranque por red).
  2. Oculta opciones de arranque manuales
    • Desactiva "Option key Display" y "Boot device List F12 Option" para que no puedan tocar el orden de arranque sin acceso a la UEFI.
  3. Establece y bloquea el orden de arranque
    • Configura el orden de dispositivos para arrancar primero desde disco interno.
    • Activa "Boot Order Lock" (si está disponible) para impedir que se cambie sin entrar a la configuración protegida.

Una vez aplicados todos los ajustes de bastionado, es fundamental guardar la configuración antes de reiniciar el sistema:

  1. Revisa que todos los parámetros modificados estén correctamente configurados.
  2. Utiliza la opción "Save and Exit" (normalmente con la tecla F10) para guardar los cambios y reiniciar el equipo.
  3. No apagues el sistema antes de guardar; los cambios se perderían.
  4. Tras reiniciar, verifica que las opciones estén activas y los cambios persistan.

Recuerda que, aunque el bastionado UEFI es una medida de seguridad efectiva, cualquier acceso físico al equipo puede permitir la manipulación o borrado de la configuración de la BIOS/UEFI, anulando estas protecciones. Por eso, es fundamental complementar la seguridad lógica con controles físicos y auditorías periódicas.
