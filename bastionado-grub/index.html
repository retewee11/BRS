En este codelab explicaremos de manera detallada el paso a paso necesario para bastionar el gestor de arranque GRUB en un sistema Debian 13.1. El objetivo es reforzar la seguridad del proceso de arranque, evitando accesos no autorizados y protegiendo la integridad del sistema desde las primeras etapas del inicio.

Editar:

sudo nano /etc/default/grub

Parámetros recomendados:

GRUB_TIMEOUT=0
GRUB_DISABLE_RECOVERY=true

GRUB_TIMEOUT se usa para que al iniciar el sistema no salga el menú del GRUB

GRUB_DISABLE_RECOVERY se usa para no tener modo recovery en el menú de GRUB

Aplicar cambios:

sudo update-grub

Generar Contraseña Cifrada

sudo grub-mkpasswd-pbkdf2

Copiar la salida (hash).

Añadir Contraseña en /etc/grub.d/40_custom

sudo nano /etc/grub.d/40_custom

Añadir al final:

set superusers="root"
password_pbkdf2 root HASH_AQUI

Guardar y aplicar cambios:

sudo update-grub

sudo chmod 700 /etc/grub.d/40_custom

los permisos 700 significan que el propietario tiene permisos totales en el archivo y el resto de usuarios no tienen permisos ninguno.

Reiniciar y comprobar:

sudo reboot

El menú debe solicitar contraseña al editar o arrancar en modo de recuperación.
